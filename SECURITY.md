# Security Policy

SHAMS is a scientific and engineering software project.

## Reporting a vulnerability
If you believe you have found a security vulnerability:
1. Do **not** publish exploit details in a public issue.
2. Open a GitHub issue titled **SECURITY: <short summary>** with minimal reproduction detail, or follow the contact path described in `GOVERNANCE.md`.

## Scope
Typical security reports may include:
- dependency vulnerabilities
- path traversal / unsafe file writes
- untrusted input execution
- evidence pack tampering risks

Scientific correctness issues should be reported as standard bugs with minimal reproductions.
